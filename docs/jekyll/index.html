Query your dependencies.


NOSQL movement is not only about key value stores... so why not taking advantage of a graph database like neo4j to show and query the dependencies of your project?

Or even better, all the dependencies of all the projects of your organization?

If you are using sbt...

Last version

The artifacts have been uploaded to maven central, so you can just add the plugin with:

groupID: com.github.dzsessona
artifactID: sbt-neo-dependencies
revision: 1.0.0

Last change
